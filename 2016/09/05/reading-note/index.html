笔记

###元字符
元字符就是在正则表达式里有特殊含义的字符，所以这些字符就无法用来代表它们本身。任何一个元字符都可以通过给它加上一个()作为前缀的办法来转义。书最后的附录提供了完整的转义字符表，这里记录一些

元字符

\d 任何一个数字字符[0-9]
\D 任何一个非数字字符[^0-9]
\w 任何一个字母、数字或下划线字符[a-zA-Z0-9_]
\W ^\w
\s 任何一个空白字符
\S 任何一个非空白字符

  • 一个或多个字符
  • 零个或多个字符 放在字符或字符集后面就可以匹配该字符（或字符集合）连续出现零次或多次的情况
    ？ 零个或一个字符

表格涉及了匹配次数，这个是相当重要的~

###重复匹配

为了防止过度匹配，引入两个概念：贪婪型元字符、懒惰型元字符

贪婪型元字符：*和+都是贪婪型元字符，会在匹配的时候尽可能的从一段文本的开头一直匹配到这段的末尾

懒惰型元字符：懒惰在这里的意思是匹配尽可能少的字符串，写法很简单，只要给贪婪型元字符加上？后缀即可。

###边界

单词边界：\bword\b匹配单词 匹配一个完整的单词
字符串边界： ^开头 $结尾
注意： \b匹配的是一个位置，这个位置位于一个能够用来构成单词的字符和一个不能用来构成单词的字符之间。

###子表达式

子表达式是一个更大的表达式的一部分：把一个表达式划分为一系列子表达式的目的是为了把那些子表达式当作一个独立元素来使用。子表达式必须用()括起来。
把所有的子表达式都加上括号的话对匹配没有任何影响，对匹配速度可能会有点影响（摘自书里，没有测试过）

e.g.
匹配四位年份：19|20\d{2} 是错误的 => (19|20)\d{2}

子表达式的常见用途：
对重复次数的元字符的作用对象做出精确的设定和控制，对|操作符的or条件作出准确的定义。

###回溯引用：前后一致匹配(backreference)

我感觉回溯引用还是要好好理解一下的。回溯引用是子表达式的另一个重要应用，允许正则表达式模式引用前面的匹配结果。

  • 末尾\1代表前面第一个子表达式
  • e.g. [ ]+(\w)[ ]+\1匹配两个单词连续重复出现2次
  • js用\来标识回溯引用，\和$配合进行替换操作时例外，也就是说在js中想要替换前面所匹配出来的用$1|2……

###javascript正则相关

/g 全局匹配，/i 忽略大小写，这两个可以组合成gi

string.prototype.replace(str, newStr | function)

变量名 代表的值

$$ |插入一个 “$”。
$& |插入匹配的子串。
$` |插入当前匹配的子串左边的内容。
$’ |插入当前匹配的子串右边的内容。
$n |假如第一个参数是 RegExp对象，并且 n 是个小于100的非负整数，那么插入第 n 个括号匹配的字符串。

如果第二个参数是函数，该函数的参数见MDN链接

string.prototype.match(regexp)

当字符串匹配到正则表达式（regular expression）时，match() 方法会提取匹配项。
return 一个包含匹配结果的数组，没有的话则返回null
